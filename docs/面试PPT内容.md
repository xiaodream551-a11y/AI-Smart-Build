## 第 2 页 | 项目背景与痛点

**幻灯片正文**

> 核心问题：BIM 建模效率低、门槛高、重复劳动多

| 痛点           | 现状描述                                          |
| -------------- | ------------------------------------------------- |
| 重复劳动量大   | 一栋 5 层框架包含数百个构件，逐个手动放置耗时数天 |
| 参数修改成本高 | 方案调整需逐一修改截面、位置，容易遗漏出错        |
| 技术门槛高     | Revit 操作复杂、菜单层级深，新手上手周期长        |

**一句话概括**
"能不能用一句话，就让 Revit 自动完成建模？"

**演讲备注**

- 结合实际经历：在课程设计中手动建一栋框架花了多少时间，改方案又花了多少时间。
- 强调这不是个别现象，是整个行业的效率瓶颈。
- 自然过渡到下一页："针对这些痛点，我设计了 AI 智建系统。"

## 第 3 页 | 解决方案概览

**幻灯片正文**

> AI 智建 = 参数化一键生成 + AI 自然语言建模

```
┌──────────────────────────────────────────┐
│            AI 智建 · 双模式架构            │
├───────────────────┬──────────────────────┤
│   参数化面板       │    AI 对话面板        │
│  填表 → 一键生成   │  说中文 → AI 解析     │
│  整栋框架结构      │  → 自动执行建模       │
├───────────────────┴──────────────────────┤
│         共享建模引擎（统一内核）            │
├──────────────────────────────────────────┤
│              Revit API                   │
└──────────────────────────────────────────┘
```

- **模式一 · 参数化面板**：输入层数、跨数、截面等参数 → 一键生成完整框架
- **模式二 · AI 对话面板**：用中文下达指令 → DeepSeek 大模型解析 → 自动执行
- **关键设计**：两种模式共享同一套建模引擎，功能只需实现一次

**演讲备注**

- 解释为什么要做"双模式"：参数化适合批量生成，AI 对话适合灵活修改，互为补充。
- 强调"共享引擎"的设计思想，这是整个系统的核心架构决策。

## 第 4 页 | 系统架构

**幻灯片正文**

> 四层架构，两个入口共享同一个引擎

```
┌─────────────────────────────────────────┐
│  表现层   参数化面板（WPF）│ AI 对话面板  │
├─────────────────────────────────────────┤
│  智能解析层       DeepSeek 大模型 API    │
│            Prompt + 16 Few-shot 示例     │
│            JSON 指令解析 + 错误恢复       │
├─────────────────────────────────────────┤
│  建模引擎层  create_column │ create_beam  │
│             create_floor  │ create_grid  │
│             modify_element│delete_element│
├─────────────────────────────────────────┤
│  Revit API 层    pyRevit / Revit 2024   │
└─────────────────────────────────────────┘
```

**技术栈一览**

| 组件       | 技术选型         | 选型理由                |
| ---------- | ---------------- | ----------------------- |
| BIM 平台   | Revit 2024       | 行业主流                |
| 开发框架   | pyRevit + Python | 快速迭代、生态丰富      |
| 大模型 API | DeepSeek         | 中文能力强、成本低      |
| 数据处理   | openpyxl / JSON  | 对接设计院 Excel 工作流 |
| 界面       | WPF              | 原生 Windows 体验       |

**演讲备注**

- 重点讲"两个入口、一个引擎"的好处：代码复用、维护成本低、新增功能自动两端可用。
- 简单提一下 DeepSeek 的选型理由：中文理解能力强，API 价格是 GPT-4 的几十分之一。

## 第 6 页 | 核心功能演示 — AI 对话建模

**幻灯片正文**

> 用中文对话，驱动 Revit 自动建模

**对话示例**

| 用户输入                          | 系统执行              |
| --------------------------------- | --------------------- |
| "在 A1 交点放一根 500×500 的柱子" | 解析 → 创建柱构件     |
| "把刚才那根梁截面改成 300×700"    | 解析 → 修改梁属性     |
| "删掉第3层的所有柱子"             | 解析 → 批量删除       |
| "帮我生成一栋5层框架"             | 解析 → 调用参数化引擎 |

**处理流程**
用户中文输入 → DeepSeek 大模型 → JSON 结构化指令 → 建模引擎执行

**技术保障**

- 16 个 Few-shot 示例确保解析准确率
- 多轮对话上下文记忆
- 9 种指令动作全覆盖：创建、修改、删除、查询、框架生成

**演讲备注**

- 现场演示或播放录屏效果最好：输入一句话，看 Revit 里构件自动出现。
- 解释 Few-shot 的作用：给大模型看 16 个"标准答案"，让它学会输出格式。
- 提到多轮对话："把刚才那根梁改大一号"这种指代，系统能理解。

## 第 7 页 | 技术亮点

**幻灯片正文**

> 四大技术亮点

**1. Prompt 工程**

- 精心设计的系统提示词，严格约束 JSON 输出格式
- 16 个 Few-shot 示例覆盖全部 9 种指令动作
- 确保大模型输出可被程序直接解析执行

**2. 离线测试体系**

- 289 个自动化测试用例，100% 通过
- 引擎层与 Revit API 解耦：Mac 上开发，Windows 上运行
- 每次提交自动回归，保障代码质量

**3. 错误恢复机制**

- API 调用超时自动重试
- JSON 解析失败自动修复重试
- 全程中文友好提示，不让用户看到报错堆栈

**4. UI 设计系统**

- 深蓝 + 橙色双色图标体系，16 个图标全部自研
- 4 倍超采样抗锯齿，确保 Revit 工具栏清晰显示
- 6 个 WPF 窗口统一视觉主题

**演讲备注**

- Prompt 工程是 AI 功能的核心，可以简单展示一段 Prompt 片段。
- 离线测试体系是工程化亮点：强调"不依赖 Revit 就能跑全部测试"。
- 错误恢复：大模型不是 100% 可靠，工程化的容错设计是必要的。
- UI 设计说明你不只是写代码，还考虑了用户体验。

## 第 8 页 | 项目创新点

**幻灯片正文**

> 四个维度的创新

**创新一 · 大模型 × BIM 跨界融合**
将大语言模型引入 BIM 建模流程，国内本科阶段的前沿探索
不是简单调用 API，而是构建了完整的"自然语言 → 结构化指令 → 建模执行"管线

**创新二 · 双模式统一架构**
参数化生成与 AI 对话共享同一套建模引擎
新增功能只需实现一次，两个入口自动可用

**创新三 · 面向实际工作流**
Excel 参数导入直接对接设计院现有流程
不是实验室原型，而是考虑了真实使用场景

**创新四 · 低门槛 BIM 交互**
用自然语言替代复杂的菜单操作
降低 BIM 建模的学习和使用门槛

**演讲备注**

- 创新一最重要，详细展开：目前国内将 LLM 与 Revit 结合的研究很少，本科阶段更少。
- 创新二体现架构设计能力，可以对比"如果两套模式各写一套引擎会怎样"。
- 创新三展现工程思维：不是做完就放着，而是考虑实际落地。
- 创新四回扣痛点页，形成叙事闭环。
