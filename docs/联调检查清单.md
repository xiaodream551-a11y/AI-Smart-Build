# AI 智建联调检查清单

## 1. Mac 离线检查

先确认纯 Python 逻辑没有回归：

```bash
python3 -m pytest
```

检查 AI JSON 提取和楼层映射：

```bash
python3 tools/debug_ai_floor.py
```

批量回归检查 AI 回复样例：

```bash
python3 tools/run_ai_regression.py
```

调试指定回复文本：

```bash
python3 tools/debug_ai_floor.py --reply '```json
{"action":"modify_section","params":{"element_type":"column","floor":2,"old_section":"400x400","new_section":"500x500"}}
```'
```

使用自定义标高序列调试：

```bash
python3 tools/debug_ai_floor.py --levels '±0.000,F1,F2,屋面' --reply '```json
{"action":"create_column","params":{"x":6000,"y":0,"base_floor":2,"top_floor":3,"section":"500x500"}}
```'
```

## 2. DeepSeek 配置检查

推荐方式：

1. 复制示例配置文件
2. 填入真实 API Key
3. 放到用户目录配置路径

macOS 路径：

```text
~/.ai-smart-build/config.json
```

示例文件：

```text
examples/config.example.json
```

也可以使用环境变量：

```bash
export DEEPSEEK_API_KEY='your-real-key'
export DEEPSEEK_MODEL='deepseek-chat'
```

## 3. Windows / Revit 联调前准备

需要准备：

1. Revit 2024
2. pyRevit 4.8+
3. 已加载结构柱、结构梁、楼板类型
4. 已把包含 `AI智建.extension` 的目录加入 pyRevit 扩展路径
5. 已执行 pyRevit Reload

Windows 用户配置路径：

```text
C:\Users\<你的用户名>\.ai-smart-build\config.json
```

## 4. Revit 内重点验证项

### 一键生成

检查项：

1. 进度条能完整走到结束
2. 生成统计和日志文件路径正常显示
3. 轴网、标高、柱、梁、板数量与参数一致

### AI 对话

最小回归用例：

1. 创建首层柱
2. 创建二层梁
3. 创建二层楼板
4. 修改二层柱截面
5. 删除三层梁
6. 统计首层柱数量

重点确认：

1. `create_column` 的 `base_floor/top_floor` 是否按边界编号解释
2. 梁、板、修改、删除、统计中的 `floor` 是否按故事层号解释
3. `/help`、`/reset`、`/retry`、`/replay`、`/replaylog`、`/replayfail` 是否可用
4. `/retry` 是否会重新请求 AI，`/replay` 是否会直接重放上一条归一化指令
5. `/replaylog` 是否能从最近一次会话文件读取并重放最后一条指令，并优先使用同名 JSON
6. `/replayfail` 是否只列出失败轮次，且列表中能看到失败原因摘要和恢复建议摘要
7. `/replayfail` 在失败动作超过一种时，是否会先按动作筛选再列出轮次
8. `/replayfail` 在失败来源超过一种时，是否会先按来源筛选，能区分普通 AI 失败、`/replaylog` 失败、`/replayfail` 二次失败
9. 输入 `/replayfail 楼层/截面/未找到` 这类关键字后，是否会先按失败原因摘要和恢复建议摘要筛选结果
10. 失败列表中是否提供“上一条”“下一条”导航项，并能沿当前筛选结果连续切换
11. 失败动作筛选和失败轮次列表是否按最近失败优先显示，且动作名是否为中文可读形式
12. 失败轮次的 Markdown/JSON 会话记录里是否包含“恢复建议”
13. 执行阶段若只返回“楼层超出范围/缺少参数/未找到目标”等失败文本，是否也会被记录成失败轮次
14. 对话结束后是否同时导出 Markdown 和 JSON 会话日志

### Excel 导入

检查项：

1. 首层柱是否落在 `±0.000 -> F1`
2. 二层梁是否落在 `F2`
3. 非法楼层和空截面是否被正确跳过
4. 日志中是否记录跳过行

### 修改与删除

检查项：

1. 单个修改和批量修改都能返回明确结果
2. 单个删除和批量删除都能返回明确结果
3. 操作完成后会导出日志

## 5. 日志检查

默认日志目录：

```text
~/Documents/AI智建日志/
```

在 Windows 上会展开为当前用户的 `Documents\AI智建日志\`。

建议检查：

1. 文件是否成功生成
2. 摘要计数是否正确
3. 详情是否包含具体动作和时间戳
4. AI 对话是否额外生成 Markdown 会话记录，且包含原始回复与归一化指令
5. 会话记录中是否包含动作统计、成功/失败数量和 AI 请求耗时
