# AI SmartBuild Integration Testing Checklist

## 1. Mac Offline Checks

First confirm that pure Python logic has no regressions:

```bash
python3 -m pytest
```

Check AI JSON extraction and floor mapping:

```bash
python3 tools/debug_ai_floor.py
```

Batch regression check on AI reply samples:

```bash
python3 tools/run_ai_regression.py
```

Debug a specific reply text:

````bash
python3 tools/debug_ai_floor.py --reply '```json
{"action":"modify_section","params":{"element_type":"column","floor":2,"old_section":"400x400","new_section":"500x500"}}
```'
````

Debug with a custom level sequence:

````bash
python3 tools/debug_ai_floor.py --levels '±0.000,F1,F2,屋面' --reply '```json
{"action":"create_column","params":{"x":6000,"y":0,"base_floor":2,"top_floor":3,"section":"500x500"}}
```'
````

## 2. DeepSeek Configuration Check

Recommended approach:

1. Copy the example configuration file
2. Fill in your real API key
3. Place it in the user directory configuration path

macOS path:

```text
~/.ai-smart-build/config.json
```

Example file:

```text
examples/config.example.json
```

You can also use environment variables:

```bash
export DEEPSEEK_API_KEY='your-real-key'
export DEEPSEEK_MODEL='deepseek-chat'
```

## 3. Windows / Revit Pre-Integration Preparation

Requirements:

1. Revit 2024
2. pyRevit 4.8+
3. Structural column, structural beam, and floor slab types loaded
4. The directory containing `AISmartBuild.extension` added to pyRevit extension path
5. pyRevit Reload executed

Windows user configuration path:

```text
C:\Users\<your-username>\.ai-smart-build\config.json
```

## 4. Key Verification Items in Revit

### One-Click Generation

Checklist:

1. Progress bar runs to completion
2. Generation statistics and log file path display correctly
3. Grid, level, column, beam, and slab counts match the parameters

### AI Chat

Minimum regression cases:

1. Create a first-floor column
2. Create a second-floor beam
3. Create a second-floor slab
4. Modify second-floor column cross-section
5. Delete third-floor beams
6. Query the number of first-floor columns

Key confirmations:

1. `create_column`'s `base_floor/top_floor` is interpreted as boundary numbers
2. `floor` in beam, slab, modification, deletion, and statistics is interpreted as story floor number
3. `/help`, `/reset`, `/retry`, `/replay`, `/replaylog`, `/replayfail` are functional
4. `/retry` re-sends the request to the AI; `/replay` directly replays the last normalized command
5. `/replaylog` reads and replays the last command from the most recent session file, preferring the companion JSON
6. `/replayfail` lists only failed turns, and the list shows failure reason summary and recovery suggestion summary
7. `/replayfail` filters by action first when there is more than one type of failed action, then lists turns
8. `/replayfail` filters by source first when there is more than one failure source, distinguishing regular AI failures, `/replaylog` failures, and `/replayfail` secondary failures
9. Entering `/replayfail 楼层/截面/未找到` (keywords) filters by failure reason summary and recovery suggestion summary first
10. The failure list provides "previous" and "next" navigation items for sequential switching within the current filter results
11. Failed action filtering and failed turn lists are shown most-recent-first, with Chinese-readable action names
12. Failed turn Markdown/JSON session records include "recovery suggestions"
13. If the execution phase only returns failure text like "floor out of range / missing parameter / target not found" without raising an exception, it is still recorded as a failed turn
14. Both Markdown and JSON session logs are exported after the conversation ends

### Excel Import

Checklist:

1. First-floor columns land on `±0.000 -> F1`
2. Second-floor beams land on `F2`
3. Invalid floors and empty cross-sections are correctly skipped
4. Skipped rows are recorded in the log

### Modification & Deletion

Checklist:

1. Both single and batch modifications return clear results
2. Both single and batch deletions return clear results
3. A log is exported after operations complete

## 5. Log Verification

Default log directory:

```text
~/Documents/AI智建日志/
```

On Windows, this expands to the current user's `Documents\AI智建日志\`.

Recommended checks:

1. Files are generated successfully
2. Summary counts are correct
3. Details include specific actions and timestamps
4. AI chat additionally generates a Markdown session record containing raw replies and normalized commands
5. Session records include action statistics, success/failure counts, and AI request timing
