## 一、系统架构

系统采用四层架构设计，自上而下分别为表现层、智能解析层、建模引擎层和 Revit API 层。各层职责清晰，通过标准接口进行通信，既保证了功能的独立性，也便于测试与维护。

```
+------------------------------------------------------------------+
|                         表现层 (UI)                                |
|   pyRevit 工具栏按钮 + WPF 窗口 (XAML)                             |
+------------------+-------------------+---------------------------+
                   |                   |
    +------ 参数表单输入 ------+    自然语言输入
                   |                   |
+------------------+     +-------------+---------------------------+
|   建模引擎层 (Engine)  |     |           智能解析层 (AI)              |
|   frame_generator     |     |  DeepSeekClient -> Parser -> 分发    |
|   grid / level /      |     +------------------------------------+
|   column / beam /     |                   |
|   floor / modify /    | <----- JSON 指令 --+
|   export / logger     |
+------------------+----+
                   |
+------------------+---------------------------------------------------+
|                        Revit API 层                                   |
|   pyRevit 框架 -> .NET API -> Document / Transaction / Collector      |
+----------------------------------------------------------------------+
```

### 1.1 表现层 (UI)

表现层负责用户交互，基于 pyRevit 工具栏与 WPF 窗口实现。

**面板布局**（共 5 个面板、8 个按钮）：

| 构件操作 | 修改构件   | 修改已有构件的截面、位置等属性             |
| 构件操作 | 删除构件   | 按条件筛选并删除构件                       |

### 1.2 智能解析层 (AI)

智能解析层负责将用户的自然语言输入解析为结构化 JSON 指令。

**核心模块**：

- **DeepSeekClient** (`ai/client.py`)：封装 HTTP 请求，管理多轮对话历史（最多 20 轮），支持自动重试与超时控制。在 IronPython 环境下使用 `System.Net.HttpWebRequest` 发送请求，在 CPython 环境下回退到标准库。
- **System Prompt** (`ai/prompt.py`)：定义 JSON Schema 输出约束，包含 16 个 Few-shot 示例，覆盖全部 10 种 action 类型，引导大模型输出合法的 JSON 指令。
- **Parser** (`ai/parser.py`)：从 API 返回文本中提取 JSON，进行参数归一化（如截面 `"500"` 补全为 `"500x500"`、楼层范围校验等），并将指令分发到对应的引擎函数。
- **Recovery** (`ai/recovery.py`)：解析失败时尝试修复常见的 JSON 格式问题（如尾逗号、缺少引号等）。
- **Replay** (`ai/replay.py`)：支持指令重放，用于调试和回归测试。
- **ConversationParser** (`ai/conversation_parser.py`)：对话流解析辅助模块。
- **ChatController** (`ai/chat_controller.py`)：对话流程控制，衔接 UI 层与 AI 解析层。

**支持的 action 类型（共 10 种）**：

| action           | 说明             | 主要参数                                                                      |
| ---------------- | ---------------- | ----------------------------------------------------------------------------- |
| `create_column`  | 创建柱           | x, y, base_floor, top_floor, section                                          |
| `create_beam`    | 创建梁           | start_x, start_y, end_x, end_y, floor, section                                |
| `create_slab`    | 创建楼板         | boundary, floor                                                               |
| `modify_section` | 修改截面         | element_type, floor, old_section, new_section                                 |
| `delete_element` | 删除构件         | element_type, floor, position                                                 |
| `generate_frame` | 生成整栋框架     | x_spans, y_spans, num_floors, floor_height, column_section, beam_section_x 等 |
| `query_count`    | 查询构件数量     | element_type, floor                                                           |
| `query_detail`   | 查询构件明细     | element_type, floor, section                                                  |
| `query_summary`  | 查询模型统计汇总 | floor                                                                         |
| `unknown`        | 无法识别的指令   | --                                                                            |

### 1.3 建模引擎层 (Engine)

建模引擎层是系统的核心计算层，封装了全部 Revit 建模操作。该层不直接依赖 pyRevit 的 `DB` 模块，而是通过参数注入的方式接收 Revit Document 对象，从而保证可在无 Revit 环境下进行离线测试。

| 模块       | 文件                 | 职责                                                       |
| ---------- | -------------------- | ---------------------------------------------------------- |
| 框架生成器 | `frame_generator.py` | 编排完整的框架生成流程：轴网 -> 标高 -> 逐层生成柱、梁、板 |
| 轴网       | `grid.py`            | 创建 X 向（数字编号）和 Y 向（字母编号）轴网线             |
| 标高       | `level.py`           | 创建各层标高（支持首层特殊层高）                           |
| 柱         | `column.py`          | 在指定坐标放置矩形截面柱                                   |
| 梁         | `beam.py`            | 在两点间创建矩形截面梁                                     |
| 楼板       | `floor.py`           | 按边界曲线创建楼板                                         |
| 修改与删除 | `modify.py`          | 单个或批量修改构件属性、删除构件                           |
| 构件查询   | `element_utils.py`   | 共享的构件属性查询函数，供 parser 和 export 共用           |
| 数据导出   | `export.py`          | 支持 JSON 和 Excel 双格式导出模型数据                      |
| 日志系统   | `logger.py`          | 操作日志记录与会话日志管理                                 |

### 1.4 Revit API 层

通过 pyRevit 框架调用 Autodesk Revit 的 .NET API。所有模型修改操作必须在 `Transaction` 事务内执行，以保证数据一致性。主要使用的 API 对象包括：

| API 对象                   | 用途                     |
| -------------------------- | ------------------------ |
| `Document`                 | 当前 Revit 模型文档      |
| `Transaction`              | 事务管理，保证原子性操作 |
| `FilteredElementCollector` | 按条件查询模型中的构件   |
| `FamilyInstance`           | 柱、梁等族实例           |
| `Floor`                    | 楼板对象                 |
| `Grid`                     | 轴网线                   |
| `Level`                    | 标高                     |

## 二、核心算法

### 2.1 参数化框架生成算法

参数化框架生成是系统的核心功能之一，能够根据用户输入的结构参数自动生成完整的钢筋混凝土框架结构模型。

**输入参数**：

| 参数                 | 类型        | 说明                 | 示例                 |
| -------------------- | ----------- | -------------------- | -------------------- |
| `x_spans`            | list[float] | X 向各跨跨距（mm）   | `[6000, 6000, 6000]` |
| `y_spans`            | list[float] | Y 向各跨跨距（mm）   | `[6000, 6000]`       |
| `num_floors`         | int         | 层数                 | `5`                  |
| `floor_height`       | float       | 标准层高（mm）       | `3600`               |
| `first_floor_height` | float       | 首层层高（mm），可选 | `4200`               |
| `column_section`     | str         | 柱截面               | `"500x500"`          |
| `beam_section_x`     | str         | X 向梁截面           | `"300x600"`          |
| `beam_section_y`     | str         | Y 向梁截面，可选     | `"300x600"`          |

**算法流程**：

```
第一步：计算轴线坐标
  x_coords = cumsum([0] + x_spans)   例如 [0, 6000, 12000, 18000]
  y_coords = cumsum([0] + y_spans)   例如 [0, 6000, 12000]

第二步：创建轴网
  X 向轴网：沿 X 轴方向，编号 1, 2, 3, ...（共 len(x_coords) 条）
  Y 向轴网：沿 Y 轴方向，编号 A, B, C, ...（共 len(y_coords) 条）
  每条轴网线超出柱中心 GRID_EXTENSION(1500mm)

第三步：创建标高
  F1 = 0 mm（基础标高）
  F2 = first_floor_height（首层顶标高）
  F3 = F2 + floor_height
  F(n+1) = F(n) + floor_height
  共创建 num_floors + 1 个标高

第四步：逐层生成构件（从第 1 层到第 num_floors 层）
  4.1 柱：在所有轴线交点 (x_i, y_j) 放置
      底标高 = 当前层标高，顶标高 = 上一层标高
  4.2 X 向梁：连接同一 Y 轴线上相邻的 X 轴交点
      例如 (x_1, y_j) -> (x_2, y_j)
  4.3 Y 向梁：连接同一 X 轴线上相邻的 Y 轴交点
      例如 (x_i, y_1) -> (x_i, y_2)
  4.4 楼板：在每个梁围合的矩形区域内生成楼板
```

**输出**：返回统计字典，包含各类构件的创建数量。

### 2.2 自然语言解析流程

自然语言解析是 AI 智建的另一核心功能，通过大语言模型将中文建模需求转化为可执行的结构化指令。

**完整流程**：

```
用户中文输入
    |
    v
附加 System Prompt（JSON Schema 定义 + 16 个 Few-shot 示例）
    |
    v
发送至 DeepSeek API（HTTP POST）
    |
    v
API 返回 JSON 字符串
    |
    v
parse_command() 提取 JSON
    |
    +---> 解析成功 ---> 参数归一化
    |                       |
    |                       v
    |                   指令分发（根据 action 路由到 engine 函数）
    |                       |
    |                       v
    |                   执行建模操作
    |
    +---> 解析失败 ---> recovery 尝试修复
                            |
                            +---> 修复成功 ---> 回到参数归一化
                            |
                            +---> 修复失败 ---> 返回错误提示
```

**参数归一化规则**：

- 截面补全：`"500"` -> `"500x500"`
- 楼层范围校验：确保楼层编号在有效范围内
- 坐标单位统一：默认使用毫米（mm）
- 缺省参数填充：未指定的参数使用系统默认值

**错误恢复策略**（`recovery.py`）：

- 去除 markdown 代码块包裹（如 ` ```json ... ``` `）
- 修复尾部多余逗号
- 补全缺失的引号
- 处理单引号替换为双引号

### 2.3 多轮对话管理

系统支持多轮对话，能够在上下文中理解用户的连续指令。

**实现机制**：

- 维护 `conversation[]` 列表，包含 `system`、`user`、`assistant` 三种角色的消息
- 每轮对话自动追加 `user` 消息和 `assistant` 回复
- 当消息轮数超过 `MAX_CONVERSATION_TURNS`（默认 20 轮）时，自动裁剪最早的对话记录，保留 system prompt
- 用户输入 `/reset` 命令可清空对话历史，开始新会话
- 用户输入 `/undo` 命令可撤销上一轮对话

## 三、API 调用方式

### 3.1 DeepSeek LLM API

系统通过 HTTP POST 请求调用 DeepSeek 大语言模型 API，将自然语言转换为结构化 JSON 指令。

**请求配置**：

| 配置项       | 值                                             |
| ------------ | ---------------------------------------------- |
| URL          | `https://api.deepseek.com/v1/chat/completions` |
| 方法         | POST                                           |
| 模型         | `deepseek-chat`                                |
| 认证方式     | Bearer Token（API Key）                        |
| Content-Type | `application/json`                             |
| 普通请求超时 | 30000 ms                                       |
| 框架生成超时 | 60000 ms                                       |
| 重试次数     | 2 次                                           |
| 重试退避系数 | 1.5                                            |

**请求体格式**：

```json
{
  "model": "deepseek-chat",
  "messages": [
    { "role": "system", "content": "<System Prompt>" },
    { "role": "user", "content": "在A1轴交点处创建一根柱子" },
    { "role": "assistant", "content": "{...}" },
    { "role": "user", "content": "<当前用户输入>" }
  ]
}
```

**响应处理**：

1. 从响应 JSON 中提取 `choices[0].message.content`
2. 调用 `parse_command()` 解析为结构化指令
3. 解析失败时触发 `recovery` 模块尝试修复

**IronPython 兼容性**：

由于 pyRevit 运行在 IronPython 环境下，HTTP 请求使用 .NET 的 `System.Net.HttpWebRequest`，而非 Python 标准库的 `urllib` 或第三方库 `requests`。代码中通过 `try/except` 实现双运行时兼容。

### 3.2 Revit API（通过 pyRevit）

所有建模操作通过 pyRevit 框架调用 Revit 的 .NET API，核心操作均在 `Transaction` 事务中执行。

**事务使用模式**：

```python
t = Transaction(doc, u"创建柱")
t.Start()
try:
    # 执行建模操作
    t.Commit()
except Exception:
    t.RollBack()
```

**常用 API 调用**：

| 操作     | API 方法                                                                    |
| -------- | --------------------------------------------------------------------------- |
| 查询构件 | `FilteredElementCollector(doc).OfClass(...)`                                |
| 创建柱   | `doc.Create.NewFamilyInstance(point, symbol, level, StructuralType.Column)` |
| 创建梁   | `doc.Create.NewFamilyInstance(curve, symbol, level, StructuralType.Beam)`   |
| 创建楼板 | `doc.Create.NewFloor(curveArray, floorType, level, ...)`                    |
| 创建轴网 | `Grid.Create(doc, line)`                                                    |
| 创建标高 | `Level.Create(doc, elevation)`                                              |
| 修改参数 | `element.LookupParameter(name).Set(value)`                                  |
| 删除构件 | `doc.Delete(elementId)`                                                     |

**默认族名称**（中文 Revit 2024 模板）：

| 构件类型 | 族名称           |
| -------- | ---------------- |
| 柱       | `混凝土-矩形-柱` |
| 梁       | `混凝土-矩形梁`  |
| 楼板     | `常规 - 150mm`   |

## 五、配置说明

### 5.1 配置文件

配置文件路径：`~/.ai-smart-build/config.json`

```json
{
  "DEEPSEEK_API_KEY": "sk-xxxxxxxxxxxxxxxx",
  "DEEPSEEK_MODEL": "deepseek-chat",
  "DEEPSEEK_API_URL": "https://api.deepseek.com/v1/chat/completions",
  "API_TIMEOUT_MS": 30000,
  "FRAME_API_TIMEOUT_MS": 60000,
  "API_RETRY_COUNT": 2,
  "MAX_CONVERSATION_TURNS": 20
}
```

### 5.2 配置优先级

系统支持环境变量覆盖配置文件中的值，优先级从高到低为：

```
环境变量 > 配置文件 (~/.ai-smart-build/config.json) > 代码默认值
```

### 5.3 配置项说明

| 配置项                   | 环境变量别名  | 默认值                                         | 类型   | 说明                         |
| ------------------------ | ------------- | ---------------------------------------------- | ------ | ---------------------------- |
| `DEEPSEEK_API_KEY`       | `LLM_API_KEY` | （空）                                         | string | DeepSeek API 密钥，必须配置  |
| `DEEPSEEK_MODEL`         | `LLM_MODEL`   | `deepseek-chat`                                | string | 调用的模型名称               |
| `DEEPSEEK_API_URL`       | `LLM_API_URL` | `https://api.deepseek.com/v1/chat/completions` | string | API 请求地址                 |
| `API_TIMEOUT_MS`         | --            | `30000`                                        | int    | 普通请求超时时间（毫秒）     |
| `FRAME_API_TIMEOUT_MS`   | --            | `60000`                                        | int    | 框架生成请求超时时间（毫秒） |
| `API_RETRY_COUNT`        | --            | `2`                                            | int    | 请求失败重试次数             |
| `API_RETRY_BACKOFF`      | --            | `1.5`                                          | float  | 重试退避系数                 |
| `MAX_CONVERSATION_TURNS` | --            | `20`                                           | int    | 最大对话轮数                 |

### 5.4 Revit 默认参数

### 5.5 启动校验

系统启动时自动执行配置校验（`config.validate_config()`），检查以下项目：

1. API Key 是否为空或仍为占位符
2. API URL 是否以 `http://` 或 `https://` 开头
3. 模型名称是否为空

校验不通过时返回警告信息，不阻断系统运行。

## 六、测试体系

### 6.1 测试概述

项目包含 **289 个离线测试**，全部可在 Mac/Linux 环境下运行，无需 Revit 或 Windows 依赖。测试框架使用 pytest。

- 模拟 `FilteredElementCollector` 的查询行为
- 模拟 `FamilyInstance`、`Floor`、`Grid`、`Level` 等对象
- 模拟 `Transaction` 的 `Start()`/`Commit()`/`RollBack()` 生命周期
- 模拟构件参数的 `LookupParameter().Set()` 操作

## 八、开发规范

### 8.1 编码规范

| 规范项         | 要求                                         |
| -------------- | -------------------------------------------- |
| 自然语言输出   | 全部使用简体中文                             |
| 代码注释       | 中文或英文均可                               |
| 变量与函数命名 | 英文 snake_case                              |
| 字符串前缀     | 中文字符串使用 `u""` 前缀（IronPython 兼容） |
| 文件编码       | UTF-8                                        |

### 8.2 Revit API 规范

- 所有修改操作必须在 `Transaction` 事务内执行
- `engine/` 目录下的函数不得直接 `from pyrevit import DB`，`DB` 对象通过参数传入
- 构件查询使用 `FilteredElementCollector`，避免遍历全部元素

### 8.3 新增 action 检查清单

新增 action 类型时，必须同步更新以下 6 处：

1. **parser.py**：添加 action 别名映射
2. **recovery.py**：添加错误恢复支持
3. **logger.py**：添加日志记录格式
4. **prompt.py**：在 System Prompt 中添加 action 定义与 Few-shot 示例
5. **回归用例**：在 `examples/ai_reply_regression_cases.json` 中添加测试用例
6. **分发测试**：在测试文件中添加 action 分发验证

### 8.4 测试规范

- 每个新功能必须有对应的离线测试
- 测试必须能在 Mac/Linux 上通过 `pytest` 运行，不依赖 Revit
- 使用 `FakeDocument` 模拟 Revit 文档对象
- 回归用例文件：`examples/ai_reply_regression_cases.json`
